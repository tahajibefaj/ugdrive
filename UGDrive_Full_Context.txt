SECTION 1 — WHAT IS UG DRIVE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

UG Drive (Unlimited Google Drive) is a personal, self-hosted cloud storage
manager. One person (the owner) connects multiple Google Drive accounts and
manages all of them from a single dashboard.

Core concept: Google gives every Gmail account 15 GB free. With 6 accounts,
that's 90 GB free. The problem is managing 6 separate accounts is messy.
UG Drive pools them into one interface with smart upload routing — files
automatically go to the account with the most free space.

Target user: Tahajib Efaj (imtahajib@gmail.com), a single personal user.
This is NOT a multi-tenant SaaS. No paying users, no ads, no monetization.
Prototype phase only. Budget: $0.

SECTION 5 — DATABASE SCHEMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TABLE: users
  id           uuid PK auto-generated
  email        text unique
  password_hash text (bcrypt)
  name         text
  avatar       text (URL)
  created_at   timestamptz

TABLE: google_accounts
  id           bigserial PK
  user_id      uuid FK → users(id) cascade delete
  email        text (Gmail address)
  name         text (Google display name)
  avatar       text (Google profile picture URL)
  token_b64    text (pickle.dumps(Credentials) → base64.b64encode → stored here)
  total_bytes  bigint
  used_bytes   bigint
  synced_at    timestamptz
  UNIQUE: (user_id, email)

TABLE: file_cache
  gid          text (Google Drive file ID)
  account_id   bigint FK → google_accounts(id) cascade delete
  user_id      uuid FK → users(id) cascade delete
  name         text
  mime         text
  size         bigint
  parent_gid   text (parent folder's Drive ID, null = root)
  created_at   text
  modified_at  text
  trashed      boolean
  view_link    text (Google Drive web URL)
  PRIMARY KEY: (gid, account_id)

TABLE: reset_tokens
  id           bigserial PK
  user_id      uuid FK → users(id) cascade delete
  token        text unique (secrets.token_urlsafe(32))
  expires_at   timestamptz (1 hour from creation)
  used         boolean default false
  created_at   timestamptz

All tables have RLS DISABLED. Auth is handled server-side via JWT.

SECTION 6 — AUTHENTICATION SYSTEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

UG DRIVE ACCOUNT (platform login):
  Register: name + email + password (min 8 chars)
  Password: bcrypt hashed before storage
  Login: bcrypt.checkpw() verification
  Session: JWT HS256, 30-day expiry
    Payload: sub (user_id), email, exp, iat
    Secret: JWT_SECRET env var
  Cookie: "ugdrive_token", httpOnly, samesite=lax, max_age=30 days
  All /api/* routes call require_user() → decode cookie → return user dict

GOOGLE DRIVE CONNECTION (OAuth 2.0):
  User clicks "+ Connect Drive" → redirected to /auth/google/start
  → Google login → user authorizes → redirected to /auth/google/callback
  → credentials saved to google_accounts table in Supabase
  → back to /dashboard
  Scopes: drive (full), userinfo.email, userinfo.profile, openid
  Redirect URI: https://ugdrive-production.up.railway.app/auth/google/callback
  Token storage: pickle.dumps(creds) → base64 → token_b64 column in Supabase
  Token refresh: on each API call, if expired and refresh_token exists → refresh
                 and save updated token back to Supabase

PASSWORD RESET (no email — personal app approach):
  POST /api/auth/forgot-password {email}
  → generates token (token_urlsafe(32)), stores in reset_tokens with 1hr expiry
  → returns {reset_url: "https://ugdrive.../reset-password?token=xxx"} directly
  → user copies link, opens in browser → /reset-password page
  POST /api/auth/reset-password {token, password}
  → verifies token exists, not used, not expired
  → updates user password_hash, marks token as used

CHANGE PASSWORD (logged in):
  POST /api/auth/change-password {current_password, new_password}
  → verifies current password with bcrypt
  → updates password_hash in Supabase

SECTION 11 — FULL BUG HISTORY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BUG: Render /data permission denied
  Cause: render.yaml had disk mount at /data (paid feature), DATA_DIR was /data
  Fix: Changed DATA_DIR to ./data (local folder, no paid disk needed)
  Status: FIXED

BUG: redirect_uri_mismatch (OAuth Error 400)
  Cause 1: Old DrivePool callback path /auth/callback used in Google Console,
           new code uses /auth/google/callback
  Cause 2: BASE_URL env var not set, had trailing slash, or was http not https
  Cause 3: Google Console URI update takes up to a few hours to propagate
  Fix: Updated Google Console URI, set BASE_URL correctly, waited for propagation
  Status: FIXED

BUG: Registration failed
  Cause: SUPABASE_KEY was the new-format "Secret key" (sb_secret_...) instead of
         the legacy JWT service_role key. Supabase updated their UI to show new
         format keys by default. The Python supabase library only accepts legacy JWT.
  Fix: Go to Supabase → Settings → API → "Legacy anon, service_role API keys" tab
       → copy the service_role key (starts with eyJhbGci...)
  Status: FIXED

BUG: Google accounts disappear after redeploy
  Cause: Original code stored OAuth tokens as .pkl files in tokens/ directory on
         the server filesystem. Railway/Render wipe filesystem on every redeploy.
  Fix: Moved ALL persistent data to Supabase. Tokens stored as base64-encoded
       pickled Credentials in google_accounts.token_b64 column.
  Status: FIXED — accounts now survive all restarts indefinitely

BUG: Avatar in topbar not clickable
  Cause: <div class="av" id="av"> had no onclick attribute
  Fix: Added onclick="toggleProfileMenu()" and built full profile dropdown
  Status: FIXED (List 1)

BUG: Icons render black in dark mode
  Cause: --amber CSS variable referenced in mimeIcon() function but never defined
         in either :root[data-theme] block. SVG stroke="var(--amber)" fell back
         to black (invalid CSS variable = currentColor = inherited = #000).
         Affected: audio files (🎵), ZIP/RAR archives
  Fix: Added --amber: #d29922 to dark theme, --amber: #9a6700 to light theme
  Status: FIXED (List 1)

BUG: OAuth 403 access_denied when adding a Google account
  Cause: The Google account email was not added as a Test User in Google Cloud
         Console OAuth consent screen. Google returns error=access_denied.
         Backend callback had no error parameter — crashed with 422 Unprocessable.
  Fix: Added Optional[str] error parameter to google_callback. When error is
       access_denied, redirects to /dashboard?oauth_error=not_test_user.
       Dashboard shows a clear toast: "add this email as a Test User in Google
       Cloud Console first."
  Status: FIXED (List 1)

BUG: Preview panel not working (images show broken)
  Cause: Grid view and preview panel tried to load /api/files/{gid}/thumbnail
         but this endpoint did not exist in main.py. Returned 404.
  Fix: Added GET /api/files/{gid}/thumbnail — checks file is image/*, downloads
       from Google Drive, streams with 1-hour cache header.
  Status: FIXED (List 1)

BUG: Move feature opens modal but files don't move

SECTION 13 — DESIGN DECISIONS & RATIONALE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHY SUPABASE INSTEAD OF LOCAL SQLITE:
  Original code stored everything (SQLite db + OAuth tokens) in local filesystem.
  Railway/Render wipe filesystem on every redeploy. Users lost all connected
  Google accounts constantly. Supabase is an external persistent database — it
  survives any number of redeployments.

WHY RAILWAY OVER RENDER:
  Render free tier: sleeps after 15 min idle, ~30s wake time, filesystem wiped
  Railway free tier: never sleeps, always on, $5/month free credit (enough for
  personal use), no credit card required. Koyeb (also always-on, no CC) was
  considered but requires credit card at signup.

WHY NO EMAIL FOR PASSWORD RESET:
  Budget $0, personal app with one user. No email service configured.
  Reset link is generated and displayed directly on screen. This works fine for
  a personal app where only the owner is using it. A future improvement would
  add Resend.com (free tier: 100 emails/day) for actual email delivery.

WHY TOKENS AS PICKLE IN SUPABASE:
  Google's Python Credentials object is not JSON-serializable (contains refresh
  token, access token, expiry, scopes, etc.). pickle.dumps() captures the full
  object state. base64 encoding makes it safe for text storage in PostgreSQL.
  On every API call: load → check expiry → refresh if needed → save updated token.

WHY IBM PLEX SANS FOR DASHBOARD:
  User wanted "Replit-style font" — clean, technical, premium-feeling.
  IBM Plex Sans matches Replit's aesthetic and is free on Google Fonts.
  Login/reset pages use Geist (another Replit/Vercel-adjacent font).

WHY PREVIEW PANEL INSTEAD OF NEW PAGE:
  Slides open from the right side of the file list. Doesn't navigate away.
  Shows: file icon, name, type, size, modified/created dates, account email.
  For images: shows actual thumbnail via /api/files/{gid}/thumbnail endpoint.
  Actions: Download, Open in Drive, Rename.

UG Drive the app: fully compliant. Uses official Google Drive API with OAuth 2.0
exactly as Google intends third-party apps to work.

Using real existing accounts: fully compliant.

Creating throwaway accounts purely for storage: technically against ToS
("Don't create multiple accounts to circumvent limitations") but rarely enforced
for personal use. Risk: Google could terminate those accounts and files are lost.

UG Drive does not encourage ToS violations.

If you are continuing this project, here is what you need to know:

1. Always read the actual uploaded file before modifying. User uploads their
   current version — do not assume previous generated code is still current.

2. Supabase key confusion will come up again if a new Supabase project is created.
   Always remind: Settings → API → "Legacy anon, service_role API keys" tab.

3. Railway auto-redeploys when GitHub is updated. User updates GitHub manually
   (edit file in browser, commit). They do not use CLI or git locally.

4. Google Console OAuth URI must match BASE_URL/auth/google/callback exactly.
   Changes take up to a few hours to propagate. Don't debug for 2 minutes and
   conclude it's broken — wait.

5. The user is non-technical. Explain every step explicitly. Never say
   "update the config" without saying exactly which file, which line, what value.

6. Budget is $0. All services must have a free tier that works without CC.
   Railway (no CC), Supabase (no CC), Google Cloud (free APIs), UptimeRobot (free).

7. The project is prototype-phase. Performance optimizations and edge cases
   are lower priority than getting features working.

8. Main pain points historically:
   - Google tokens disappearing → always Supabase
   - redirect_uri_mismatch → always check BASE_URL env var + Google Console URI
   - Registration failed → always check Supabase legacy key
